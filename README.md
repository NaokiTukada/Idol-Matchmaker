# 推しネーター

## 1. 概要 (Overview)
ユーザーのパーソナルな質問への回答（推し活スタンスや好み）に基づき、事前に定義された38人の日本の有名アイドルの中から、最も相性の良い「推すべきアイドル」を推薦するMBTI風の診断Webアプリケーション。

## 2. 背景・想定ユーザー (Background & Target Users)

### 背景
日本のアイドル界は「戦国時代」と言われるほど選択肢が膨大で、嵐・モー娘・AKBのような全国区の「天下」グループはもはや存在しない。複数のグループやメンバーに目移りしやすく、グループ単位（曲・ダンス・文化）での好きはあっても、**推しを1人に絞れない**という悩みが生まれやすい。

本アプリは、そうした「推しを誰かに決めてほしい」というニーズから生まれた。

### 想定ユーザー
* **推し活初心者〜中級者** — 最近アイドルにハマり始め、情報過多で迷っている人
* **グループ推し寄りの人** — グループ全体の魅力は分かるが、メンバー単位の推しが定まらない人
* **決断を後押ししてほしい人** — 自分の好みを言語化し、客観的な指標で「この人を推せ」と示してほしい人

### 解決したい課題
* 選択肢が多すぎて推しを1人決められない
* 推し活をしてみたいが、自分では決められない
* 好みは漠然としているが、自分に合う推しの軸が分からない
* 推し活をもっと深く楽しみたいが、熱量を注ぐ対象が見つからない

## 3. システム要件 (System Requirements)
* **Frontend:** TypeScript (フレームワーク未定: React/Next.js 等を想定)
* **Backend:** FastAPI (Python)
* **Architecture:** 軽量3層アーキテクチャ (Router / Service / Schema)
    * 初期開発速度とFastAPIの特性を活かすため、軽量な構成を採用。
    * 診断アルゴリズムの変更容易性を担保するため、スコア計算部分はStrategyパターンで分離する。

## 4. 診断アルゴリズム (Algorithm)
**多次元ベクトルマッチング（距離計算）**を採用する。

1.  ユーザーの回答から6次元のベクトル（各軸 -1.0 〜 1.0 のスコア）を生成。
2.  事前に定義された38人のアイドルの6次元ベクトルマスターデータと照合。
3.  ベクトル間の距離（類似度）を計算し、最も距離が近い（数値の形が似ている）アイドルをリターンする。

## 5. 評価軸 (Evaluation Metrics)
独自の6軸（6次元）を用いて、ユーザーの志向とアイドルの特性をマッピングする。

| 軸 | -1.0 寄り | 1.0 寄り | 備考（役割） |
| :--- | :--- | :--- | :--- |
| **1. 関係性** | 育成・親近感 | 心酔・カリスマ | 距離感、どう関わりたいか |
| **2. パフォーマンス**| 王道・アイドル性 | 独自性・アーティスト | 表現の方向性 |
| **3. ビジュアル** | 可愛い・ポップ | 美人・クール | 見た目の直接的な好み |
| **4. キャラクター** | わちゃわちゃ | 孤高・スター性 | 性格の距離感 |
| **5. 年齢属性** | 年下感（守りたい） | 年上感（頼りたい） | 精神的な依存の方向性 |
| **6. 振る舞い** | 天然・マイペース | しっかり・知的 | 思考・行動の特性 |
